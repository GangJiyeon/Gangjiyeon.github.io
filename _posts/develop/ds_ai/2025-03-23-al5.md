# Greedy Algorithm
## Greedy 알고리즘의 개념
**[ 개념 - Local Optimization Approach ]**  
> 지엽적인 방법을 가고 최적화된(라고 믿는)값을 선택하는 방법
> • 일련의 선택들을 수행하며, 각 선택은 그 순간 가장 좋아 보이는 것을 선택함  
> • 각 선택은 **국소적으로 최적**일 수 있으나, **전역적으로 최적**은 아닐 수 있음

**[ 예시: Exchanges Greedy Algorithm ]**

```c
while (there are more coins and the instance is not solved){
    grab the largest remaining coin;            // selection procedure
    if (adding the coin makes the change exceed the amount owed)  // feasibility check
        reject the coin;
    else
        add the coin to the change;
    if (the total value of the change equals the amount owed)     // solution check
        the instance is solved;
}
```

# Divide and Conquer 
## Divide and Conquer (분할 정복) 소개

- **Top-Down 접근 방식**  
  → 문제 인스턴스를 더 작은 인스턴스로 나누고  
  → 더 작은 문제들의 해를 결합하여 원래 문제를 해결함

---

## 🔹 2. 이진 탐색 (Binary Search)의 원리

- **정의:**  
  정렬된 배열에서 특정 키 \( x \)가 있는지를 탐색

- **기본 원리:**  
  1. 배열의 **중간 원소**를 찾는다  
  2. \( x \)가 중간 값보다 작으면 **왼쪽 부분 배열**로  
     크면 **오른쪽 부분 배열**로 이동  
  3. 이 과정을 반복 (재귀)

- **Divide and Conquer 구조에 대응되는 방식**  
  - **Divide:** 배열을 두 부분으로 나눈다  
  - **Conquer:** 해당 부분에 대해 재귀 수행  
  - **Obtain:** 하위 문제의 결과가 곧 전체 문제의 결과  
    (이진 탐색은 결과 결합 필요 없음)

> ⚠️ 이진 탐색은 출력 결합(combination of outputs)이 필요 없는 특별한 분할 정복 알고리즘

---

## 🔹 3. 이진 탐색 알고리즘 설계

- **재귀 설계 절차**
  - (1) 더 작은 인스턴스로 나누는 방법을 정의
  - (2) 종료 조건(terminal condition)을 명확히 설정
  - (3) 종료 조건에 도달했을 때 결과를 반환

- **꼬리 재귀 사용**
  - 재귀 호출 이후 추가 연산 없음
  - 반복문으로 쉽게 변환 가능 → 메모리 효율 증가

- 📌 **스택 깊이**는 재귀 호출 깊이에 따라 결정됨  
  → 깊이가 클수록 메모리 사용량 증가

---

## 🔹 4. 재귀 관계와 시간 복잡도 분석

### ▶ Worst-Case Time Complexity

- 재귀 관계식:

\[
W(n) = W(n/2) + 1
\]

→ 하나의 비교 + 하위 문제 \( n/2 \)에 대한 재귀 호출

- 재귀 함수로 표현:

\[
t_n = t_{n/2} + 1, \quad t_1 = 1 \quad (\text{단, } n = 2^k)
\]

- 몇 가지 값으로 살펴보면:

| \( n \) | \( t_n \) 계산 과정     | 결과 |
|--------|--------------------------|------|
| 2      | \( t_1 + 1 \)             | 2    |
| 4      | \( t_2 + 1 = 2 + 1 \)     | 3    |
| 8      | \( t_4 + 1 = 3 + 1 \)     | 4    |
| 16     | \( t_8 + 1 = 4 + 1 \)     | 5    |

⇒ 즉, \( t_n = \log_2 n + 1 \)

---

### ▶ 귀납적 증명 (Induction)

- **기본 단계:**

\[
t_1 = 1 = \log_2 1 + 1
\]

- **귀납 단계:**  
  \( t_{2n} = t_n + 1 = \log_2 n + 1 + 1 = \log_2 (2n) + 1 \)

# 📚 Mergesort 알고리즘 (순서 재구성 + 번역문 유지)

## ✅ 1. Mergesort 2 (in-place-sort)

### 🔹 Mergesort 2

**문제:**  
\( n \)개의 키를 비내림차순으로 정렬하라.

**입력:**  
양의 정수 \( n \), 키 배열 \( S \), 인덱스는 1부터 \( n \)까지

**출력:**  
비내림차순으로 정렬된 키를 담고 있는 배열 \( S \)

```c
void mergesort2 (index low, index high)
{
  index mid;

  if (low < high) {
    mid = ⌊(low + high)/2⌋;
    mergesort2(low, mid);
    mergesort2(mid + 1, high);
    merge2(low, mid, high);
  }
}
```

---

### 🔹 Merge 2

**문제:**  
Mergesort 2에서 생성된 \( S \)의 두 정렬된 부분 배열을 병합하라.

**입력:**  
인덱스 \( low \), \( mid \), \( high \), 배열 \( S \)의 구간 \( [low, high] \)  
→ \( [low, mid] \)와 \( [mid+1, high] \)는 이미 정렬되어 있음

**출력:**  
\( S \)의 부분 배열 \( [low, high] \)이 비내림차순으로 정렬된 상태

```c
void merge2 (index low, index mid, index high)
{
  index i, j, k;
  keytype U[low..high];  // 병합을 위한 지역 배열

  i = low; j = mid + 1; k = low;
  while (i ≤ mid && j ≤ high) {
    if (S[i] < S[j]) {
      U[k] = S[i];
      i++;
    }
    else {
      U[k] = S[j];
      j++;
    }
    k++;
  }

  if (i > mid)
    move S[j] through S[high] to U[k] through U[high];
  else
    move S[i] through S[mid] to U[k] through U[high];

  move U[low] through U[high] to S[low] through S[high];
}
```

## ✅ 2. Mergesort 기본 버전 (with extra space)

### 🔹 Mergesort

**문제:**  
\( n \)개의 키를 비내림차순으로 정렬하라.

**입력:**  
양의 정수 \( n \), 키 배열 \( S \), 인덱스는 1부터 \( n \)까지

**출력:**  
비내림차순으로 정렬된 키를 담고 있는 배열 \( S \)

```c
void mergesort (int n, keytype S[])
{
  if (n > 1) {
    const int h = ⌊n / 2⌋, m = n - h;
    keytype U[1..h], V[1..m];
    copy S[1] through S[h] to U[1] through U[h];
    copy S[h+1] through S[n] to V[1] through V[m];
    mergesort(h, U);
    mergesort(m, V);
    merge(h, m, U, V, S);
  }
}
```

> *→ 병렬적인 공간을 필요로 한다.*

---

### 🔹 Merge

**문제:**  
정렬된 두 배열을 하나의 정렬된 배열로 병합하라.

**입력:**  
양의 정수 \( h \), \( m \), 정렬된 배열 \( U[1..h] \), \( V[1..m] \)

**출력:**  
\( U \)와 \( V \)의 키를 포함한, 1부터 \( h + m \)까지 인덱싱된 정렬된 배열 \( S \)

```c
void merge (int h, int m, const keytype U[],
            const keytype V[], keytype S[])
{
  index i, j, k;
  i = 1; j = 1; k = 1;
  while (i <= h && j <= m) {
    if (U[i] < V[j]) {
      S[k] = U[i];
      i++;
    }
    else {
      S[k] = V[j];
      j++;
    }
    k++;
  }

  if (i > h)
    copy V[j] through V[m] to S[k] through S[h+m];
  else
    copy U[i] through U[h] to S[k] through S[h+m];
}
```

> *→ Inplace sort가 아니다 (추가 공간을 만들어 사용함)*

**Mergesort Algorithm Example**

- 입력: `27, 10, 12, 20, 25, 13, 15, 22`
